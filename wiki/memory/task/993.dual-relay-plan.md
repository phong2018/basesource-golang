# Task 993 — Dual Relay: Kafka + RabbitMQ Outbox

## Problem

The current outbox uses a **single relay** (`OutboxRelay`) with one `IEventPublisher` and a single `published_at` / `status` on `outbox_events`.  
When two brokers are required (Kafka for domain event streaming, RabbitMQ for notification task queue), a single status field is not enough:

- Kafka success + RabbitMQ failure → `published_at` is set → RabbitMQ delivery is silently lost.
- No way to retry only the failed broker.
- No independent monitoring per destination.

---

## Solution: `outbox_events` + `outbox_deliveries`

Split the current single-table design into two tables.

### `outbox_events` (unchanged shape, drop status/published_at)

```sql
-- +goose Up
CREATE TABLE outbox_events (
    id             BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    event_id       VARCHAR(36)  NOT NULL UNIQUE,
    event_type     VARCHAR(128) NOT NULL,
    aggregate_type VARCHAR(64)  NOT NULL,
    aggregate_id   VARCHAR(64)  NOT NULL,
    payload        JSON         NOT NULL,
    created_at     DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### `outbox_deliveries` (new)

```sql
CREATE TABLE outbox_deliveries (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    outbox_event_id BIGINT UNSIGNED NOT NULL,
    destination     VARCHAR(32)  NOT NULL,   -- 'kafka' | 'rabbitmq'
    status          VARCHAR(16)  NOT NULL DEFAULT 'pending',
    attempt_count   INT UNSIGNED NOT NULL DEFAULT 0,
    last_error      TEXT,
    published_at    DATETIME(3),
    created_at      DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at      DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    CONSTRAINT fk_od_event FOREIGN KEY (outbox_event_id) REFERENCES outbox_events(id),
    INDEX idx_od_dest_status (destination, status, id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Transactional write (inside usecase)

```sql
BEGIN;
INSERT INTO todos ...;
INSERT INTO outbox_events (event_id, event_type, aggregate_type, aggregate_id, payload)
    VALUES ('evt-uuid', 'todo.created', 'todo', '42', '{"title":"Buy milk"}');
INSERT INTO outbox_deliveries (outbox_event_id, destination, status)
    VALUES (LAST_INSERT_ID(), 'kafka',    'pending'),
           (LAST_INSERT_ID(), 'rabbitmq', 'pending');
COMMIT;
```

```go
type IOutboxRepository interface {
    // write path (called inside usecase transaction)
    CreateEventWithDeliveries(ctx context.Context, event *model.OutboxEvent, destinations []string) error

    // relay read path — scoped per destination
    ListPendingDeliveries(ctx context.Context, destination string, limit int) ([]*model.OutboxDelivery, error)
    GetEventByID(ctx context.Context, id uint) (*model.OutboxEvent, error)

    // relay update path
    MarkDeliveryPublished(ctx context.Context, deliveryID uint) error
    MarkDeliveryFailed(ctx context.Context, deliveryID uint, errMsg string) error
}
```

## Infrastructure Layer Changes

### `outbox_repository_impl.go`

| Method | SQL |
|--------|-----|
| `CreateEventWithDeliveries` | TX: INSERT event → get last_insert_id → INSERT N delivery rows |
| `ListPendingDeliveries` | `SELECT * FROM outbox_deliveries WHERE destination=? AND status='pending' ORDER BY id LIMIT ?` |
| `GetEventByID` | `SELECT * FROM outbox_events WHERE id=?` |
| `MarkDeliveryPublished` | `UPDATE outbox_deliveries SET status='published', published_at=NOW(), attempt_count=attempt_count+1 WHERE id=?` |
| `MarkDeliveryFailed` | `UPDATE outbox_deliveries SET status='failed', last_error=?, attempt_count=attempt_count+1 WHERE id=?` |

### `outbox_relay.go` — destination-aware relay

One generic struct, two named constructors. The broker name never appears as a magic string outside this file.

```go
type OutboxRelay struct {
    repo        domainRepo.IOutboxRepository
    publisher   domainService.IEventPublisher
    destination string
}

// Named constructors — call site reads as self-documenting code.
func NewKafkaOutboxRelay(repo domainRepo.IOutboxRepository, publisher domainService.IEventPublisher) *OutboxRelay {
    return &OutboxRelay{repo: repo, publisher: publisher, destination: model.OutboxDestinationKafka}
}

func NewRabbitMQOutboxRelay(repo domainRepo.IOutboxRepository, publisher domainService.IEventPublisher) *OutboxRelay {
    return &OutboxRelay{repo: repo, publisher: publisher, destination: model.OutboxDestinationRabbitMQ}
}

func (r *OutboxRelay) process(ctx context.Context) {
    deliveries, _ := r.repo.ListPendingDeliveries(ctx, r.destination, relayBatchSize)
    for _, d := range deliveries {
        event, _ := r.repo.GetEventByID(ctx, d.OutboxEventID)
        if err := r.publisher.Publish(ctx, event); err != nil {
            slog.ErrorContext(ctx, "outbox relay: publish failed", "destination", r.destination, "delivery_id", d.ID, "error", err)
            r.repo.MarkDeliveryFailed(ctx, d.ID, err.Error())
            continue
        }
        r.repo.MarkDeliveryPublished(ctx, d.ID)
    }
}
```

## Integration Script Updates Required

`outbox_events.status` and `outbox_events.published_at` no longer exist after this change.
Any script that queries those columns must be updated to query `outbox_deliveries` instead.

### `994.script-test-kafka`

**Affected:** `test_01_create.sh`, `test_02_update.sh`, `test_03_delete.sh`

The `wait_for_db_status` helper queries:
```sql
SELECT status FROM outbox_events WHERE aggregate_id=? AND event_type=?
```

Replace with:
```sql
SELECT d.status
FROM outbox_events e
JOIN outbox_deliveries d ON d.outbox_event_id = e.id
WHERE e.aggregate_id = '$agg'
  AND e.event_type   = '$event_type'
  AND d.destination  = 'kafka'
ORDER BY d.id DESC LIMIT 1;
```

`test_04_replay.sh` — no DB status query, no change needed.

## Runtime Example

After `todo.created` is written:

| outbox_event_id | destination | status    |
|-----------------|-------------|-----------|
| evt-1           | kafka       | pending   |
| evt-1           | rabbitmq    | pending   |

Kafka relay succeeds, RabbitMQ broker is temporarily down:

| outbox_event_id | destination | status    |
|-----------------|-------------|-----------|
| evt-1           | kafka       | published |
| evt-1           | rabbitmq    | pending   |

RabbitMQ relay retries on next tick — fully independent, zero impact on Kafka.
