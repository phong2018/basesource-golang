### Transaction mock approach

Since `WithinTransaction` takes a `func(ctx) error`, the mock can simply call the fn directly:

```go
type transactionMock struct{}

func (t *transactionMock) WithinTransaction(ctx context.Context, fn func(context.Context) error) error {
    return fn(ctx)
}
```

This lets tests verify the business logic without needing a real DB.

## Step 2 — `.golangci.yml`

```yaml
linters:
  enable:
    - errcheck       # unchecked errors
    - gosimple       # simplification suggestions
    - govet          # go vet checks
    - ineffassign    # unused assignments
    - staticcheck    # static analysis
    - unused         # unused code
    - gofmt          # formatting
    - goimports      # import ordering
    - misspell       # typos in comments/strings
    - noctx          # http calls without context

linters-settings:
  gofmt:
    simplify: true

issues:
  exclude-rules:
    - path: "_test.go"
      linters:
        - errcheck
```
