## Step 8: Consumer Group Replay Test

Verify that `StartOffset: kafka.FirstOffset` works — the consumer replays all past events when the group offset is reset.

```bash
# Kill worker
pkill -f basesource-996

# Reset consumer group offsets to earliest
docker compose -f docker/docker-compose.yaml exec kafka \
  /opt/kafka/bin/kafka-consumer-groups.sh \
  --bootstrap-server localhost:9092 \
  --group todo-worker \
  --topic todo-events \
  --reset-offsets --to-earliest --execute
```

Expected output:
```
GROUP        TOPIC        PARTITION  NEW-OFFSET
todo-worker  todo-events  0          0
todo-worker  todo-events  1          0
todo-worker  todo-events  2          0
```

```bash
# Restart worker
/tmp/basesource-996 worker > /tmp/996-worker.log 2>&1 &
sleep 5
cat /tmp/996-worker.log
```

Expected: all past `domain event received` entries replayed from offset 0.

## Troubleshooting

### Kafka consumer not receiving messages after first run

**Cause:** Consumer group already exists with committed offsets at `LastOffset`. New messages published before the consumer started are missed.

**Fix:**
```bash
pkill -f basesource-996
docker compose -f docker/docker-compose.yaml exec kafka \
  /opt/kafka/bin/kafka-consumer-groups.sh \
  --bootstrap-server localhost:9092 \
  --group todo-worker --topic todo-events \
  --reset-offsets --to-earliest --execute
# Restart worker
```

### Offset reset fails with "group is inactive but current state is PreparingRebalance"

**Cause:** Worker is still running and holding the consumer group open.

**Fix:** Kill the worker first (`pkill -f basesource-996`), then run the reset command.

### outbox_events rows stuck at `status=pending`

**Cause:** Kafka broker unreachable. OutboxRelay will retry on next 2s poll cycle automatically when Kafka recovers.

**Check:** `docker compose -f docker/docker-compose.yaml ps kafka`
