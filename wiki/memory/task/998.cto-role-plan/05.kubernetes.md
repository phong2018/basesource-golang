# Task 05 — Kubernetes Manifests

## Requirement
CTO must demonstrate: basic Kubernetes operations — Deployment, Service, ConfigMap, HPA, resource limits.

## What the source currently has
- `docker/Dockerfile` — builds the Go binary
- `docker/docker-compose.yaml` — local dev only

## What to add

### Directory structure
```
k8s/
├── namespace.yaml
├── configmap.yaml          # non-secret config
├── secret.yaml             # template only (actual values via CI/CD)
├── deployment.yaml         # app Deployment with resource limits
├── service.yaml            # ClusterIP service
├── ingress.yaml            # Ingress with TLS
└── hpa.yaml                # HorizontalPodAutoscaler
```

### Key manifests

**deployment.yaml**
```yaml
spec:
  replicas: 2
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 0       # zero-downtime deploy
      maxSurge: 1
  containers:
    - name: app
      resources:
        requests: { cpu: 100m, memory: 128Mi }
        limits:   { cpu: 500m, memory: 256Mi }
      readinessProbe:
        httpGet: { path: /health, port: 8080 }
      livenessProbe:
        httpGet: { path: /health, port: 8080 }
      envFrom:
        - configMapRef: basesource-config
        - secretRef: basesource-secret
```

**hpa.yaml**
```yaml
spec:
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
```

**ingress.yaml**
- TLS termination via cert-manager / Let's Encrypt annotation
- Nginx ingress controller

## Demonstrates to CTO interviewer
- Knows Deployment vs Service vs Ingress roles
- Rolling update for zero-downtime deploy
- Resource requests/limits (prevents noisy-neighbor on shared cluster)
- HPA for horizontal scaling based on CPU
- Health probes: readiness (remove from LB if not ready) vs liveness (restart if stuck)
- Separates config (ConfigMap) from secrets (Secret) — no secrets in Git
