# Task 04 — CI/CD Pipeline (GitHub Actions)

## Requirement
CTO must demonstrate: CI/CD setup — GitHub Actions, automated testing, linting, build, and deploy pipeline.

## What the source currently has
- `Makefile` with `make check` (vet + test + lint) — runs locally only
- No `.github/workflows/` directory

## What to add

### Pipeline: 3 stages — CI → Build → Deploy

#### Stage 1: CI (`ci.yml`) — on every push / PR
```yaml
jobs:
  ci:
    - go vet ./...
    - golangci-lint run ./...
    - go test ./internal/... -v -race -count=1 -coverprofile=coverage.out
    - upload coverage artifact
```

#### Stage 2: Build & Push (`build.yml`) — on push to main
```yaml
jobs:
  build:
    - docker build -t $ECR_REGISTRY/$IMAGE:$SHA .
    - docker push
    - (optional) trivy scan image for vulnerabilities
```

#### Stage 3: Deploy (`deploy.yml`) — on push to main after build passes
```yaml
jobs:
  deploy:
    - kubectl set image deployment/basesource app=$IMAGE:$SHA
    # OR: helm upgrade --install
```

### Files to create
```
.github/
└── workflows/
    ├── ci.yml          # lint + test on every PR
    ├── build.yml       # docker build + push on merge to main
    └── deploy.yml      # kubectl rollout on merge to main
```

### GitHub Actions secrets required (document in README)
```
DOCKER_REGISTRY      # e.g. ECR URI
DOCKER_USERNAME
DOCKER_PASSWORD
KUBE_CONFIG          # base64-encoded kubeconfig for kubectl
```

### Coverage badge
- Add `go test -coverprofile` + upload to Codecov or GitHub artifact
- Add badge to `README.md`

### Caching
- Cache Go modules: `actions/cache` on `~/go/pkg/mod`
- Cache Docker layers: `docker/build-push-action` with `cache-from`

## Files to create / modify
- `.github/workflows/ci.yml`
- `.github/workflows/build.yml`
- `.github/workflows/deploy.yml`
- `README.md` — add CI badge, pipeline diagram, required secrets

## Demonstrates to CTO interviewer
- Has set up CI/CD for a team (not just knows it exists)
- Separate CI (every PR) vs deploy (main only) stages
- Security: secrets via GitHub Secrets, not hardcoded
- Artifact promotion: same image SHA deployed, not rebuilt
- Developer experience: PR author sees lint/test failures before merge
