## What to add

### 1. Index strategy in migrations
Add a new migration: `20260523000003_add_indexes.sql`

```sql
-- +goose Up
-- todos: filter by done status (common query pattern from ListTodoInput.Done)
ALTER TABLE todos ADD INDEX idx_todos_done (done);

-- todos: full-text search on title (ListTodoInput.Search)
ALTER TABLE todos ADD FULLTEXT INDEX idx_todos_title_fulltext (title);

-- todos: list sorted by created_at DESC (default list ordering)
ALTER TABLE todos ADD INDEX idx_todos_created_at (created_at DESC);

-- audit_logs: lookup by entity + entity_id (find all actions on a todo)
ALTER TABLE audit_logs ADD INDEX idx_audit_entity (entity, entity_id);

-- audit_logs: time-range queries for reporting
ALTER TABLE audit_logs ADD INDEX idx_audit_created_at (created_at);

-- +goose Down
ALTER TABLE todos DROP INDEX idx_todos_done;
ALTER TABLE todos DROP INDEX idx_todos_title_fulltext;
ALTER TABLE todos DROP INDEX idx_todos_created_at;
ALTER TABLE audit_logs DROP INDEX idx_audit_entity;
ALTER TABLE audit_logs DROP INDEX idx_audit_created_at;
```

### 2. Connection pool configuration
Update `database/database.go`:
```go
db.SetMaxOpenConns(cfg.Database.MaxOpenConns)     // default: 25
db.SetMaxIdleConns(cfg.Database.MaxIdleConns)     // default: 10
db.SetConnMaxLifetime(cfg.Database.ConnMaxLifetime) // default: 5 * time.Minute
db.SetConnMaxIdleTime(cfg.Database.ConnMaxIdleTime) // default: 1 * time.Minute
```
Config from env:
```
DB_MAX_OPEN_CONNS=25
DB_MAX_IDLE_CONNS=10
DB_CONN_MAX_LIFETIME=5m
DB_CONN_MAX_IDLE_TIME=1m
```
Document why: too many conns → MySQL OOM; too few → request queuing latency spikes.
