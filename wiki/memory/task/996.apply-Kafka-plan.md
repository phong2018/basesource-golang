## Why both brokers simultaneously

| | RabbitMQ | Kafka |
|---|---|---|
| Mental model | **Task queue** ‚Äî do this job now | **Event log** ‚Äî this thing happened |
| Message lifecycle | Deleted after ACK | Retained for days/weeks ‚Äî replayable |
| Ordering | Per-queue FIFO | Per-partition, guaranteed by key |
| Throughput | ~50k msg/s | >1M msg/s |
| Use case | Send email, push notification, webhook | Audit log, analytics, rebuild read model |
| Replay | No | Yes ‚Äî reset offset to re-process history |

**In this source:**

```
POST /api/v1/todos
  ‚îî‚îÄ usecase.Create()
       ‚îî‚îÄ WithinTransaction()
            ‚îú‚îÄ INSERT todos
            ‚îú‚îÄ INSERT audit_logs
            ‚îî‚îÄ INSERT outbox_events   ‚Üê Kafka relay picks this up
                                        durable domain event log, replayable

       ‚îî‚îÄ (after commit, non-fatal)
            ‚îî‚îÄ notifPublisher.PublishNotification(task)
                  ‚Üì
               RabbitMQ queue: todo.notifications
                  ‚Üì
               notification worker ‚Üí send email / push
```

```go
package messaging

import (
    "github.com/segmentio/kafka-go"
    "github.com/yourname/go-clean-base/config"
)

// NewKafkaWriter creates a Kafka producer.
// Hash balancer: same Key (aggregate_id) always routes to same partition ‚Üí ordering.
func NewKafkaWriter(cfg *config.MessagingConfig) *kafka.Writer {
    return &kafka.Writer{
        Addr:         kafka.TCP(cfg.KafkaBrokers...),
        Topic:        cfg.KafkaTopic,
        Balancer:     &kafka.Hash{},
        RequiredAcks: kafka.RequireAll,
        Async:        false,
    }
}

// NewKafkaReader creates a Kafka consumer for a named consumer group.
func NewKafkaReader(cfg *config.MessagingConfig) *kafka.Reader {
    return kafka.NewReader(kafka.ReaderConfig{
        Brokers:  cfg.KafkaBrokers,
        Topic:    cfg.KafkaTopic,
        GroupID:  cfg.KafkaGroupID,
        MinBytes: 1,
        MaxBytes: 10e6,
    })
}
```

**File: `internal/infrastructure/messaging/kafka_producer.go`** ‚Üê new file

Implements `domainService.IEventPublisher` (already defined in feature/997). Used by `OutboxRelay`.

```go
package messaging

import (
    "context"
    "encoding/json"
    "fmt"

    "github.com/segmentio/kafka-go"
    domainModel "github.com/yourname/go-clean-base/internal/domain/model"
    domainService "github.com/yourname/go-clean-base/internal/domain/service"
)

type kafkaProducer struct {
    writer *kafka.Writer
}

func NewKafkaProducer(writer *kafka.Writer) domainService.IEventPublisher {
    return &kafkaProducer{writer: writer}
}

func (p *kafkaProducer) Publish(ctx context.Context, event *domainModel.OutboxEvent) error {
    body, err := json.Marshal(event)
    if err != nil {
        return fmt.Errorf("marshal event: %w", err)
    }
    return p.writer.WriteMessages(ctx, kafka.Message{
        Key:   []byte(event.AggregateID), // partition key = todo ID ‚Üí ordering per todo
        Value: body,
        Headers: []kafka.Header{
            {Key: "event_type", Value: []byte(event.EventType)},
            {Key: "event_id",   Value: []byte(event.EventID)},
        },
    })
}

func (p *kafkaProducer) Close() error {
    return p.writer.Close()
}
```

**Why `Key: aggregate_id`:** `Hash` balancer routes all messages with the same key to the same partition. All events for `todo:42` (create ‚Üí update ‚Üí delete) land on the same partition, processed in order by the same consumer instance.

**File: `internal/infrastructure/messaging/kafka_consumer.go`** ‚Üê new file

```go
package messaging

import (
    "context"
    "log/slog"

    "github.com/segmentio/kafka-go"
)

type KafkaConsumer struct {
    reader *kafka.Reader
}

func NewKafkaConsumer(reader *kafka.Reader) *KafkaConsumer {
    return &KafkaConsumer{reader: reader}
}

func (c *KafkaConsumer) Start(ctx context.Context, handler func([]byte) error) error {
    for {
        // FetchMessage does NOT auto-commit ‚Äî manual commit after successful processing
        msg, err := c.reader.FetchMessage(ctx)
        if err != nil {
            if ctx.Err() != nil {
                return nil // graceful shutdown
            }
            slog.ErrorContext(ctx, "kafka fetch failed", "error", err)
            continue
        }

        if err := handler(msg.Value); err != nil {
            slog.ErrorContext(ctx, "kafka handler failed ‚Äî not committing",
                "error", err, "offset", msg.Offset, "partition", msg.Partition)
            // do NOT commit ‚Äî message re-delivered on next poll
            continue
        }

        if err := c.reader.CommitMessages(ctx, msg); err != nil {
            slog.ErrorContext(ctx, "kafka commit failed", "error", err)
        }
    }
}

// Close shuts down the Kafka reader cleanly.
func (c *KafkaConsumer) Close() error {
    return c.reader.Close()
}
```

**Why `FetchMessage` + `CommitMessages` not `ReadMessage`:** `ReadMessage` auto-commits before the handler runs ‚Äî if the handler fails the message is silently lost. Manual commit = at-least-once delivery.

### Step 7 ‚Äî RabbitMQ notification publisher

**File: `internal/infrastructure/messaging/rabbitmq_notification_publisher.go`** ‚Üê new file

Implements `domainService.INotificationPublisher`. Publishes notification tasks directly to queue `todo.notifications` via the default exchange (point-to-point ‚Äî one task, one worker):

```go
package messaging

import (
    "context"
    "encoding/json"
    "fmt"

    amqp "github.com/rabbitmq/amqp091-go"
    domainModel "github.com/yourname/go-clean-base/internal/domain/model"
    domainService "github.com/yourname/go-clean-base/internal/domain/service"
)

type notificationPublisher struct {
    ch *amqp.Channel
}

func NewNotificationPublisher(client *RabbitMQClient) domainService.INotificationPublisher {
    return &notificationPublisher{ch: client.channel}
}

func (p *notificationPublisher) PublishNotification(ctx context.Context, n *domainModel.Notification) error {
    body, err := json.Marshal(n)
    if err != nil {
        return fmt.Errorf("marshal notification: %w", err)
    }
    return p.ch.PublishWithContext(ctx,
        "",                   // default exchange ‚Äî direct-to-queue
        "todo.notifications", // routing key = queue name
        false,
        false,
        amqp.Publishing{
            ContentType:  "application/json",
            DeliveryMode: amqp.Persistent,
            Body:         body,
        },
    )
}

func (p *notificationPublisher) Close() error { return nil }
```

```go
func NewWorkerCommand() *cobra.Command {
    return &cobra.Command{
        Use:   "worker",
        Short: "Start outbox relay (Kafka) and event consumers (Kafka + RabbitMQ)",
        RunE: func(cmd *cobra.Command, args []string) error {
            logger.Setup()
            ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
            defer stop()

            cfg, err := config.NewConfig()
            if err != nil {
                return err
            }
            c, err := container.NewContainer(ctx, cfg)
            if err != nil {
                return err
            }
            defer func() {
                _ = c.DBClient.Close()
                _ = c.RabbitMQClient.Close()
                _ = c.KafkaConsumer.Close()  // closes kafka.Reader
                _ = c.KafkaProducer.Close()  // closes kafka.Writer via IEventPublisher.Close()
            }()

            g, gCtx := errgroup.WithContext(ctx)

            // goroutine 1: outbox relay ‚Üí Kafka (domain event streaming)
            g.Go(func() error {
                slog.Info("outbox relay starting (Kafka)")
                c.OutboxRelay.Start(gCtx)
                return nil
            })

            // goroutine 2: Kafka consumer ‚Äî analytics / audit log
            g.Go(func() error {
                slog.Info("kafka consumer starting",
                    "topic", cfg.Messaging.KafkaTopic,
                    "group", cfg.Messaging.KafkaGroupID)
                return c.KafkaConsumer.Start(gCtx, handleDomainEvent)
            })

            // goroutine 3: RabbitMQ notification consumer ‚Äî send email / push
            g.Go(func() error {
                slog.Info("rabbitmq notification consumer starting")
                if err := c.NotificationConsumer.Start(gCtx,
                    "todo.notifications", "todo.notifications", handleNotificationTask,
                ); err != nil {
                    return err
                }
                <-gCtx.Done()
                return nil
            })

            slog.Info("worker started")
            return g.Wait()
        },
    }
}
```

### Step 13 ‚Äî Container: wire both brokers

**File: `container/container.go`** ‚Äî replace existing content:

Note: `EventConsumer` is renamed to `NotificationConsumer` ‚Äî it now handles notification tasks (RabbitMQ), not domain events (which go to Kafka).

```go
package container

import (
    "context"

    "github.com/yourname/go-clean-base/config"
    "github.com/yourname/go-clean-base/internal/domain/service"
    "github.com/yourname/go-clean-base/internal/infrastructure/database"
    "github.com/yourname/go-clean-base/internal/infrastructure/httpclient"
    "github.com/yourname/go-clean-base/internal/infrastructure/messaging"
    infraRepo "github.com/yourname/go-clean-base/internal/infrastructure/repository"
    s3client "github.com/yourname/go-clean-base/internal/infrastructure/s3"
    "github.com/yourname/go-clean-base/internal/usecase"
)

type Container struct {
    Cfg                  *config.Config
    DBClient             *database.Client
    RabbitMQClient       *messaging.RabbitMQClient
    KafkaProducer        service.IEventPublisher      // closes kafka.Writer
    KafkaConsumer        *messaging.KafkaConsumer     // closes kafka.Reader
    NotificationConsumer *messaging.RabbitMQConsumer  // renamed from EventConsumer
    OutboxRelay          *messaging.OutboxRelay
    NotificationClient   service.INotificationClient
    S3Client             service.IFileStorage
    TodoUsecase          usecase.ITodoUsecase
}

func NewContainer(ctx context.Context, cfg *config.Config) (*Container, error) {
    db, err := database.NewClient(cfg)
    if err != nil {
        return nil, err
    }

    // RabbitMQ ‚Äî notification task queue
    rmq, err := messaging.NewRabbitMQClient(&cfg.Messaging)
    if err != nil {
        return nil, err
    }

    // Kafka ‚Äî domain event streaming
    kafkaWriter   := messaging.NewKafkaWriter(&cfg.Messaging)
    kafkaReader   := messaging.NewKafkaReader(&cfg.Messaging)
    kafkaProducer := messaging.NewKafkaProducer(kafkaWriter)
    kafkaConsumer := messaging.NewKafkaConsumer(kafkaReader)

    todoRepo      := infraRepo.NewTodoRepository(db)
    auditLogRepo  := infraRepo.NewAuditLogRepository(db)
    outboxRepo    := infraRepo.NewOutboxRepository(db)
    notifier      := httpclient.NewNotificationClient(cfg)
    s3            := s3client.NewS3Client(ctx, cfg)

    notifPublisher      := messaging.NewNotificationPublisher(rmq)
    notifConsumer       := messaging.NewRabbitMQConsumer(rmq)
    outboxRelay         := messaging.NewOutboxRelay(outboxRepo, kafkaProducer) // Kafka

    todoUsecase := usecase.NewTodoUsecase(
        todoRepo, auditLogRepo, outboxRepo, db, notifier, notifPublisher,
    )

    return &Container{
        Cfg:                  cfg,
        DBClient:             db,
        RabbitMQClient:       rmq,
        KafkaProducer:        kafkaProducer,
        KafkaConsumer:        kafkaConsumer,
        NotificationConsumer: notifConsumer,
        OutboxRelay:          outboxRelay,
        NotificationClient:   notifier,
        S3Client:             s3,
        TodoUsecase:          todoUsecase,
    }, nil
}
```

## Key concepts demonstrated

**RabbitMQ for notifications (task queue):**
- Published after commit ‚Äî real-time, low latency
- Consumed once ‚Äî one worker sends the email
- No replay ‚Äî sending the same email twice is wrong

**Kafka for domain events (event log):**
- Via outbox pattern ‚Äî durable, no event lost even if Kafka is temporarily down
- Multiple independent consumer groups each get all events
- `partition key = aggregate_id` ‚Äî all events for same todo, same partition, in-order
- Replay: reset consumer offset to re-process all historical events (impossible in RabbitMQ)

**Clean Architecture:**
- Usecase depends only on `IEventPublisher` and `INotificationPublisher` ‚Äî zero knowledge of Kafka or RabbitMQ
- Swapping either broker = change one infrastructure file + container wiring only

## Real test scenarios

1. **Happy path** ‚Äî Create todo ‚Üí Kafka receives domain event AND RabbitMQ receives notification task simultaneously
2. **Kafka down** ‚Äî API still works, outbox events queue in MySQL, catch up on recovery. RabbitMQ notifications unaffected
3. **RabbitMQ down** ‚Äî API still works (notification publish fails non-fatally, logged). Kafka domain events unaffected
4. **Ordering** ‚Äî Create + Update + Delete same todo ‚Üí all 3 Kafka events land on same partition, consumed in order
5. **Consumer group** ‚Äî run 3 workers, create 9 todos ‚Üí Kafka events distributed 3 per worker (one partition each)
6. **Replay** ‚Äî reset Kafka consumer offset to beginning, all past domain events re-consumed. RabbitMQ notifications NOT replayed (by design)
