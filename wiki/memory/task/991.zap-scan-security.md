# Task 991 — Authentication, Authorization & OWASP ZAP Security Scan

## Objective

Add JWT-based authentication and role-based authorization to the API as a purely additive feature — **no existing code is changed or removed**. Then run a thorough OWASP ZAP security scan that exercises both authenticated and unauthenticated flows.

> **Scope constraint:** All existing features (Kafka, RabbitMQ, Kubernetes, K6 load tests, outbox relay, notification workers) remain exactly as-is. Auth is wired in alongside them, not replacing anything.

---

## Part 1 — Feature: Authentication & Authorization

### Overview

Implement a standard JWT auth layer following the existing Clean Architecture rules:

```
POST /api/v1/auth/register   → create user account
POST /api/v1/auth/login      → return access + refresh token
POST /api/v1/auth/refresh    → rotate tokens
POST /api/v1/auth/logout     → revoke refresh token
GET  /api/v1/me              → current user profile (protected)

Protected resource routes require: Authorization: Bearer <token>
Role-based guard: admin can manage all todos; user can only manage their own
```

## Part 1.5 — New Protected Todo APIs (additive, for ZAP coverage)

> **Scope constraint:** Existing todo routes (`GET /todos`, `GET /todos/:id`, `POST /todos`, `PUT /todos/:id`, `DELETE /todos/:id`) and their underlying usecase/repository/domain code are **not modified**. All items below are new routes mounted on new path segments.

These endpoints give ZAP a rich authenticated surface: ownership checks, role gates, parameter injection points, file upload, and bulk operations.

### New routes overview

```
# Owner-scoped — authenticated user sees/manages only their own todos
GET    /api/v1/my/todos               list caller's todos (paginated, filtered)
GET    /api/v1/my/todos/:id           get one (403 if owned by someone else)
POST   /api/v1/my/todos               create todo owned by caller
PUT    /api/v1/my/todos/:id           update (owner or admin only)
DELETE /api/v1/my/todos/:id           soft-delete (owner or admin only)

# Bulk operations — admin only
POST   /api/v1/admin/todos/bulk-delete   delete multiple by IDs
PATCH  /api/v1/admin/todos/bulk-status   mark multiple done/undone

# Sharing — authenticated
POST   /api/v1/my/todos/:id/share        share todo with another user (by email)
DELETE /api/v1/my/todos/:id/share/:uid   revoke share

# Comments — authenticated (any user can comment on todos shared with them)
GET    /api/v1/todos/:id/comments         list comments
POST   /api/v1/todos/:id/comments         add comment
DELETE /api/v1/todos/:id/comments/:cid   delete own comment (or admin)

# Attachment — authenticated
POST   /api/v1/my/todos/:id/attachment    upload file (stores via existing S3 client)
DELETE /api/v1/my/todos/:id/attachment    remove attachment
```

## Part 2 — Updated OWASP ZAP Security Scan Plan

### Scan 2 — Authenticated full scan (active)

ZAP uses the JWT to reach protected endpoints and actively probe for injection, broken access control, etc.

Create `zap/auth-context.yaml`:
```yaml
env:
  contexts:
    - name: "API with JWT"
      urls:
        - "http://localhost:8080"
      authentication:
        method: "http"
        parameters:
          loginUrl: "http://localhost:8080/api/v1/auth/login"
          loginRequestData: '{"email":"admin@test.com","password":"Test1234!"}'
          loginPageValidation: "response.status == 200"
      sessionManagement:
        method: "httpAuthHeader"
        parameters:
          headerName: "Authorization"
          headerValue: "Bearer {%json:access_token%}"
      users:
        - name: "admin"
          credentials:
            username: "admin@test.com"
            password: "Test1234!"
        - name: "regular-user"
          credentials:
            username: "user@test.com"
            password: "Test1234!"
```

Run the authenticated scan:
```bash
docker run --rm \
  --network host \
  -v $(pwd)/zap:/zap/wrk \
  ghcr.io/zaproxy/zaproxy:stable \
  zap-full-scan.py \
  -t http://localhost:8080 \
  -z "-configfile /zap/wrk/auth-context.yaml" \
  -r /zap/wrk/zap-authenticated-report.html
```

### Scan 3 — Broken access control checks (manual)

Use `ADMIN_TOKEN` and `USER_TOKEN` captured in the seed step above.

```bash
# 401 — no token
curl -i GET http://localhost:8080/api/v1/my/todos

# 403 — user tries to access another user's todo
curl -i http://localhost:8080/api/v1/my/todos/999 \
  -H "Authorization: Bearer $USER_TOKEN"

# 403 — user tries admin bulk-delete
curl -i -X POST http://localhost:8080/api/v1/admin/todos/bulk-delete \
  -H "Authorization: Bearer $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"ids":[1,2,3]}'

# 401 — tampered JWT
curl -i http://localhost:8080/api/v1/my/todos \
  -H "Authorization: Bearer eyJtYW1wZXJlZA.payload.badsig"

# 403 — non-owner tries to delete another user's comment
curl -i -X DELETE http://localhost:8080/api/v1/todos/1/comments/5 \
  -H "Authorization: Bearer $USER_TOKEN"

# 200 — admin can bulk-delete
curl -i -X POST http://localhost:8080/api/v1/admin/todos/bulk-delete \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"ids":[1,2]}'
```

### Security test matrix

| Scenario | Endpoint | Expected | Method |
|---|---|---|---|
| No token | `GET /my/todos` | 401 | Manual |
| Expired token | any protected | 401 | Manual |
| Tampered JWT | any protected | 401 | Manual |
| `user` accesses another user's todo | `GET /my/todos/:id` | 403 | Manual |
| `user` hits admin bulk-delete | `POST /admin/todos/bulk-delete` | 403 | Manual |
| `user` hits admin bulk-status | `PATCH /admin/todos/bulk-status` | 403 | Manual |
| Non-owner deletes comment | `DELETE /todos/:id/comments/:cid` | 403 | Manual |
| SQL injection in login body | `POST /auth/login` | 401 (no crash) | ZAP active |
| SQL injection in `?search=` param | `GET /my/todos?search=` | 200 (sanitised) | ZAP active |
| SQL injection in comment body | `POST /todos/:id/comments` | 400 or 201 (no crash) | ZAP active |
| Oversized payload in comment body | `POST /todos/:id/comments` | 400 | ZAP active |
| IDOR — access todo by guessing ID | `GET /my/todos/:id` | 403 | ZAP active |
| Bulk-delete with huge ID array | `POST /admin/todos/bulk-delete` | 400 | ZAP active |
| File upload with executable MIME | `POST /my/todos/:id/attachment` | 400 | ZAP active |
| Brute-force login | `POST /auth/login` | 429 (rate limit) | ZAP active |
| Missing security headers | all routes | Medium alert | ZAP baseline |
| Sensitive data in error body | all routes | Low/Info | ZAP baseline |
| CSRF on state-changing routes | POST/PUT/DELETE | Check | ZAP baseline |
| Token returned in URL | any | High alert | ZAP baseline |
