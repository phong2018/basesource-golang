# 01 — Cluster Overview (Daily Health Check)

Run these every morning before standup. Takes ~2 minutes.

---

## Step 1 — Pod status

```bash
kubectl get pods -n basesource
```

### What you see

```
NAME                                READY   STATUS      RESTARTS   AGE
basesource-api-7f7cdcf9b-c8hc8     1/1     Running     0          25m
basesource-api-7f7cdcf9b-xfdm2     1/1     Running     0          25m
basesource-worker-577bcdbc8f-7d5hj  1/1     Running     0          25m
basesource-worker-577bcdbc8f-jjl98  1/1     Running     0          25m
fluent-bit-c8ccl                    1/1     Running     0          101m
kafka-0                             1/1     Running     0          111m
mysql-0                             1/1     Running     0          35m
node-exporter-9mfnp                 1/1     Running     0          111m
rabbitmq-0                          1/1     Running     0          39m
basesource-migrate-xmrpb            0/1     Completed   0          21m
```

### How to read

| Column | What it means |
|---|---|
| `READY 1/1` | Container is running and passed readiness probe |
| `STATUS Running` | Normal |
| `STATUS Completed` | Job finished — expected for migrate pod |
| `STATUS CrashLoopBackOff` | Container keeps crashing — check logs immediately |
| `STATUS Pending` | Pod can't be scheduled — check node resources |
| `STATUS OOMKilled` | Out of memory — increase memory limit |
| `RESTARTS > 0` | Pod has restarted — check if count is growing |

### What's a problem

```bash
# Any pod not Running or Completed
kubectl get pods -n basesource | grep -v "Running\|Completed"

# High restart count (> 3 in short time)
kubectl get pods -n basesource | awk '$5 > 3'
```

## Step 2 — HPA (autoscaler) status

```bash
kubectl get hpa -n basesource
```

### What you see

```
NAME                    TARGETS      MINPODS   MAXPODS   REPLICAS
basesource-api-hpa      cpu: 1%/70%  2         10        2
basesource-worker-hpa   cpu: 3%/70%  2         6         2
```

### How to read

| Column | Meaning |
|---|---|
| `TARGETS cpu: 1%/70%` | Current CPU is 1%, threshold is 70% — calm |
| `REPLICAS 2` | Currently 2 pods running (at minimum) |
| `REPLICAS 5` | HPA scaled up — you're under load |
| `REPLICAS 10` | At maximum — may need to raise the limit |

**Alert when:** `REPLICAS = MAXPODS` for > 5 minutes — system is at capacity.

## Step 3 — PVC (persistent storage) status

```bash
kubectl get pvc -n basesource
```

### What you see

```
NAME                       STATUS   CAPACITY   STORAGECLASS
mysql-data-mysql-0         Bound    10Gi       standard
kafka-data-kafka-0         Bound    10Gi       standard
rabbitmq-data-rabbitmq-0   Bound    5Gi        standard
```

### How to read

- `STATUS Bound` = storage is attached and healthy — normal
- `STATUS Pending` = storage provisioner is stuck — problem
- `STATUS Lost` = PV was deleted while PVC still exists — data loss risk

### Check disk usage inside pods

```bash
# MySQL disk usage
kubectl exec -n basesource mysql-0 -- df -h /var/lib/mysql

# Kafka disk usage
kubectl exec -n basesource kafka-0 -- df -h /var/lib/kafka/data

# RabbitMQ disk usage
kubectl exec -n basesource rabbitmq-0 -- df -h /var/lib/rabbitmq
```

**Alert when:** any volume > 80% full.

## Step 4 — Resource usage (requires metrics-server)

```bash
kubectl top pods -n basesource
kubectl top nodes
```

### What you see

```
NAME                                CPU(cores)   MEMORY(bytes)
basesource-api-7f7cdcf9b-c8hc8     5m           32Mi
basesource-worker-577bcdbc8f-7d5hj  12m          48Mi
mysql-0                             25m          256Mi
kafka-0                             80m          512Mi
rabbitmq-0                          15m          128Mi
```

### How to read

| Value | Meaning |
|---|---|
| `5m` CPU | 5 millicores = 0.005 of 1 CPU core — very low |
| `500m` CPU | 0.5 core — moderate |
| `1000m` CPU | 1 full core — high |
| `32Mi` memory | 32 megabytes |

**Alert when:** CPU > 80% of limit, or memory > 90% of limit consistently.

## Step 5 — StatefulSet and DaemonSet health

```bash
kubectl get statefulsets,daemonsets -n basesource
```

### What you see

```
NAME                  READY   AGE
statefulset/kafka     1/1     111m
statefulset/mysql     1/1     111m
statefulset/rabbitmq  1/1     111m

NAME                  DESIRED  CURRENT  READY
daemonset/fluent-bit  1        1        1
daemonset/node-exporter 1      1        1
```

- StatefulSet `READY 1/1` = pod is up and data is persistent
- DaemonSet `DESIRED = READY` = agent running on every node
