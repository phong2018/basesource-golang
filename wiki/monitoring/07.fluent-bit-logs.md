# 07 — Fluent Bit Log Collection

Fluent Bit runs as a DaemonSet — one pod per node. It tails all `*basesource*.log`
container log files from the node's `/var/log/containers/` and outputs parsed JSON.

---

## View collected logs

```bash
# Stream all basesource app logs collected by Fluent Bit
kubectl logs -n basesource -l app=fluent-bit -f
```

Each line is a parsed JSON log entry from any basesource pod (api or worker):

```json
{
  "date": 1780036948.13,
  "log": "2026-05-29T06:42:28Z stdout F {\"time\":\"2026-05-29T06:42:28Z\",\"level\":\"INFO\",\"msg\":\"request completed\",\"method\":\"GET\",\"path\":\"/api/v1/todos\",\"status\":200,\"latency_ms\":15}",
  "cluster": "local",
  "env": "development"
}
```

The `log` field contains the raw container log line: `<timestamp> stdout F <app-json>`.
The app's slog JSON is embedded as a string inside `log`. To extract it, parse `log` and split on ` F `:

```bash
kubectl logs -n basesource -l app=fluent-bit --tail=20 | \
  python3 -c "
import sys, json
for line in sys.stdin:
    try:
        outer = json.loads(line)
        # log field format: '2026-05-29T... stdout F {...app json...}'
        app_json = outer['log'].split(' F ', 1)[-1]
        inner = json.loads(app_json)
        print(json.dumps(inner, indent=2))
    except: pass
" 2>/dev/null | head -30
```

### Fields added by Fluent Bit

| Field | Value | Meaning |
|---|---|---|
| `date` | Unix timestamp | When Fluent Bit collected this log |
| `log` | original log line | The app's slog JSON output |
| `cluster` | `local` | Set in fluent-bit ConfigMap |
| `env` | `development` | Set in fluent-bit ConfigMap |

## Filter logs by level

```bash
# Only errors from the app (grep inside the collected output)
kubectl logs -n basesource -l app=fluent-bit -f | grep '"level":"ERROR"'

# Only slow requests (latency > 100ms)
kubectl logs -n basesource -l app=fluent-bit -f | grep '"latency_ms"' | \
  python3 -c "
import sys, json
for line in sys.stdin:
    try:
        outer = json.loads(line)
        inner = json.loads(outer['log'])
        if inner.get('latency_ms', 0) > 100:
            print(inner)
    except: pass
"
```

---

## Filter by specific path or request

```bash
# All POST /api/v1/my/todos requests
kubectl logs -n basesource -l app=fluent-bit -f | grep '"/api/v1/my/todos"' | grep '"method":"POST"'

# All 4xx/5xx responses
kubectl logs -n basesource -l app=fluent-bit -f | grep -E '"status":[45][0-9]{2}'
```

## Fluent Bit health check

```bash
# Check DaemonSet is running on all nodes
kubectl get daemonset fluent-bit -n basesource

# View Fluent Bit's own operational metrics
kubectl port-forward -n basesource daemonset/fluent-bit 2020:2020
curl http://localhost:2020/api/v1/health
curl http://localhost:2020/api/v1/metrics
```

Health response (returns version + build flags):
```json
{"fluent-bit":{"version":"3.1.10","edition":"Community",...}}
```

Metrics response (JSON — key numbers to look at):
```json
{
  "input":  {"tail.0":   {"records":28, "bytes":7880}},
  "filter": {"record_modifier.0": {"drop_records":0, "records":28}},
  "output": {"stdout.0": {"proc_records":28, "errors":0, "dropped_records":0}}
}
```

- `input.records` — total log lines collected from container log files
- `output.errors` — should be 0; > 0 means output destination is failing
- `output.dropped_records` — should be 0; > 0 means logs lost due to buffer overflow

## Fluent Bit logs (its own operational logs)

```bash
# Fluent Bit's own startup/error logs (not app logs)
kubectl logs -n basesource -l app=fluent-bit | grep -v "^{\"date\""
```

Problems to look for:
```
[error] no records in chunk   ← no app logs found — check path filter
[warn]  Mem_Buf_Limit reached ← buffering too many logs — increase limit or flush faster
```

## Changing what gets collected

Edit the ConfigMap and restart:

```bash
kubectl edit configmap fluent-bit-config -n basesource
# Change Path in [INPUT] section, e.g. to collect all containers:
# Path /var/log/containers/*.log

kubectl rollout restart daemonset/fluent-bit -n basesource
```

## Where logs go in production

The current config outputs to **stdout** (`[OUTPUT] Name stdout`).
In production, swap the output to ship to a log backend:

```ini
# Elasticsearch
[OUTPUT]
    Name  es
    Host  elasticsearch.monitoring.svc.cluster.local
    Port  9200
    Index basesource-logs

# Loki (Grafana)
[OUTPUT]
    Name  loki
    Host  loki.monitoring.svc.cluster.local
    Port  3100
    Labels job=basesource
```
