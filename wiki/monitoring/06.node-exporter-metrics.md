# 06 — Node Exporter Metrics

Node Exporter runs as a DaemonSet (one pod per node). It exposes raw Prometheus metrics
at `:9100/metrics` — hardware-level stats for the entire node.

---

## Access metrics

```bash
# Port-forward to node-exporter
kubectl port-forward -n basesource svc/node-exporter-svc 9100:9100

# Then in another terminal:
curl http://localhost:9100/metrics | head -50
```

Or target specific metrics:

```bash
curl -s http://localhost:9100/metrics | grep node_cpu
curl -s http://localhost:9100/metrics | grep node_memory
curl -s http://localhost:9100/metrics | grep node_filesystem
curl -s http://localhost:9100/metrics | grep node_network
```

---

## Key metrics and how to read them

### CPU

```bash
curl -s http://localhost:9100/metrics | grep 'node_cpu_seconds_total{.*mode="idle"'
```

```
node_cpu_seconds_total{cpu="0",mode="idle"} 12345.67
node_cpu_seconds_total{cpu="0",mode="user"} 234.56
node_cpu_seconds_total{cpu="0",mode="system"} 45.67
node_cpu_seconds_total{cpu="0",mode="iowait"} 2.34
```

These are **cumulative counters** (always increasing). To get current CPU usage %, compare two readings over time:

```
CPU usage % = 1 - (idle_delta / total_delta)
```

In practice: use `kubectl top nodes` for a quick readable number.

**Alert when:** `iowait` is consistently high → disk I/O bottleneck.

---

### Memory

```bash
curl -s http://localhost:9100/metrics | grep -E "node_memory_(MemTotal|MemAvailable|MemFree|Cached)_bytes"
```

```
node_memory_MemTotal_bytes     8589934592    ← 8 GB total
node_memory_MemAvailable_bytes 4294967296    ← 4 GB available
node_memory_MemFree_bytes      1073741824    ← 1 GB free (excludes cache)
node_memory_Cached_bytes       2147483648    ← 2 GB used as cache (reclaimable)
```

**Actual used memory = MemTotal - MemAvailable**

```bash
# Calculate used memory % quickly
curl -s http://localhost:9100/metrics | grep -E "node_memory_(MemTotal|MemAvailable)_bytes" | \
  awk '/MemTotal/{t=$2} /MemAvailable/{a=$2} END{printf "Used: %.1f%%\n", (1-a/t)*100}'
```

**Alert when:** available memory < 15% of total.

---

### Disk

```bash
curl -s http://localhost:9100/metrics | grep 'node_filesystem_avail_bytes{mountpoint="/"'
curl -s http://localhost:9100/metrics | grep 'node_filesystem_size_bytes{mountpoint="/"'
```

```
node_filesystem_size_bytes{mountpoint="/"}  107374182400   ← 100 GB total
node_filesystem_avail_bytes{mountpoint="/"}  53687091200   ← 50 GB available = 50% used
```

```bash
# Disk usage % for root filesystem
curl -s http://localhost:9100/metrics | grep 'node_filesystem_(size|avail)_bytes{.*mountpoint="/"' | \
  awk '/size/{s=$2} /avail/{a=$2} END{printf "Disk used: %.1f%%\n", (1-a/s)*100}'
```

**Alert when:** disk use > 80%.

---

### Network

```bash
curl -s http://localhost:9100/metrics | grep 'node_network_(receive|transmit)_bytes_total{device="eth0"'
```

```
node_network_receive_bytes_total{device="eth0"}   1234567890   ← bytes received since boot
node_network_transmit_bytes_total{device="eth0"}  987654321    ← bytes sent since boot
```

These are cumulative — compare two readings to get bandwidth:
```
bandwidth (bytes/s) = (current - previous) / interval_seconds
```

---

### Load average

```bash
curl -s http://localhost:9100/metrics | grep node_load
```

```
node_load1   0.45    ← 1-minute load average
node_load5   0.32    ← 5-minute
node_load15  0.28    ← 15-minute
```

Load average = number of processes waiting for CPU.
- On a 4-core node: load 1.0 = 25% utilized, load 4.0 = 100%, load > 4.0 = overloaded

**Alert when:** `node_load1 > number_of_cpu_cores`.

---

## Summary — what to check daily

```bash
kubectl port-forward -n basesource svc/node-exporter-svc 9100:9100 &

# CPU load
curl -s http://localhost:9100/metrics | grep node_load1

# Memory available
curl -s http://localhost:9100/metrics | grep node_memory_MemAvailable_bytes

# Disk available on root
curl -s http://localhost:9100/metrics | grep 'node_filesystem_avail_bytes{.*mountpoint="/"'

# Kill port-forward
pkill -f "port-forward.*9100"
```
