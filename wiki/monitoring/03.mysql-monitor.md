# 03 — MySQL Monitor

---

## Connect to MySQL

```bash
# Interactive shell
kubectl exec -it -n basesource mysql-0 -- mysql -u appuser -papppass appdb

# One-off query (non-interactive)
kubectl exec -n basesource mysql-0 -- \
  mysql -u appuser -papppass appdb -e "YOUR SQL HERE;" 2>/dev/null

# Port-forward for GUI tools (TablePlus, DBeaver, DataGrip)
kubectl port-forward -n basesource pod/mysql-0 3306:3306
# Connect: host=127.0.0.1 port=3306 user=appuser pass=apppass db=appdb
```

## Table sizes and row counts

```bash
kubectl exec -n basesource mysql-0 -- mysql -u appuser -papppass appdb -e "
SELECT
  table_name,
  table_rows,
  ROUND(data_length/1024/1024, 2) AS data_MB,
  ROUND(index_length/1024/1024, 2) AS index_MB
FROM information_schema.tables
WHERE table_schema = 'appdb'
ORDER BY data_length DESC;" 2>/dev/null
```

### How to read

```
table_name          table_rows  data_MB  index_MB
outbox_deliveries   50000       12.50    3.20
outbox_events       20000       5.00     1.10
todos               8000        1.50     0.40
users               150         0.02     0.01
refresh_tokens      300         0.05     0.02
```

- `outbox_deliveries` grows fastest — watch this table
- If `outbox_events` grows unbounded → relay workers may be stuck

## Active connections

```bash
kubectl exec -n basesource mysql-0 -- mysql -u appuser -papppass -e "SHOW STATUS LIKE 'Threads_connected';" 2>/dev/null
kubectl exec -n basesource mysql-0 -- mysql -u appuser -papppass -e "SHOW PROCESSLIST;" 2>/dev/null
```

### How to read PROCESSLIST

```
Id  User     Host           db      Command  Time  State
1   appuser  10.244.0.5     appdb   Sleep    0     —       ← idle connection
2   appuser  10.244.0.7     appdb   Query    0     —       ← active query
3   appuser  10.244.0.6     appdb   Query    45    Sending data  ← SLOW — investigate
```

- `Time > 5s` on a Query = slow query — look at `State` column
- Max connections = 500 (set via `--max-connections=500`)
- **Alert when:** `Threads_connected > 400`

## Outbox health (relay backlog)

```bash
# Pending deliveries per destination
kubectl exec -n basesource mysql-0 -- mysql -u appuser -papppass appdb -e "
SELECT destination, status, COUNT(*) as cnt
FROM outbox_deliveries
GROUP BY destination, status
ORDER BY destination, status;" 2>/dev/null
```

### How to read

```
destination  status    cnt
kafka        delivered  9800   ← normal
kafka        pending    2      ← small backlog, normal
rabbitmq     delivered  9800   ← normal
rabbitmq     pending    150    ← growing — worker may be behind
rabbitmq     failed     5      ← needs investigation
```

- `pending` should be low (< 20) in steady state
- `pending` growing = relay worker is behind Kafka/RabbitMQ
- `failed` > 0 = events that couldn't be delivered — check worker logs

## Slow query log (runtime)

```bash
kubectl exec -n basesource mysql-0 -- mysql -u appuser -papppass -e "
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;
SHOW VARIABLES LIKE 'slow_query_log%';" 2>/dev/null
```

## Disk usage

```bash
kubectl exec -n basesource mysql-0 -- df -h /var/lib/mysql
```

```
Filesystem  Size  Used  Avail  Use%
/dev/sda1   10G   1.2G  8.8G   12%   ← healthy
/dev/sda1   10G   8.5G  1.5G   85%   ← WARNING — expand PVC or archive data
```

**Alert when:** disk use > 80%.
