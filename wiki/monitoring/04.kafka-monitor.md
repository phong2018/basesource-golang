# 04 — Kafka Monitor

---

## Connect to Kafka

```bash
# All Kafka commands run inside the pod
kubectl exec -it -n basesource kafka-0 -- /bin/bash

# Or prefix each command inline:
kubectl exec -n basesource kafka-0 -- /opt/kafka/bin/<command>
```

---

## List topics

```bash
kubectl exec -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 --list
```

Expected output:
```
todo-events
__consumer_offsets
```

## Describe a topic (partitions + offsets)

```bash
kubectl exec -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-topics.sh \
  --bootstrap-server localhost:9092 \
  --describe --topic todo-events
```

### How to read

```
Topic: todo-events   PartitionCount: 3   ReplicationFactor: 1
  Partition: 0  Leader: 1  Replicas: 1  Isr: 1  Elr:  LastKnownElr:
  Partition: 1  Leader: 1  Replicas: 1  Isr: 1
  Partition: 2  Leader: 1  Replicas: 1  Isr: 1
```

- `PartitionCount: 3` — 3 partitions, up to 3 consumers can work in parallel
- `Leader: 1` — broker 1 (only 1 broker in this setup) handles all partitions
- `Isr: 1` — In-Sync Replica = 1 (matches replication factor = healthy)

## Consumer group lag (most important metric)

```bash
kubectl exec -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-consumer-groups.sh \
  --bootstrap-server localhost:9092 \
  --describe --group todo-worker
```

### How to read

```
GROUP        TOPIC        PARTITION  CURRENT-OFFSET  LOG-END-OFFSET  LAG  CONSUMER-ID
todo-worker  todo-events  0          1250            1252            2    worker-pod-1
todo-worker  todo-events  1          980             980             0    worker-pod-2
todo-worker  todo-events  2          760             762             2    worker-pod-1
```

| Column | Meaning |
|---|---|
| `CURRENT-OFFSET` | Last message the consumer has processed |
| `LOG-END-OFFSET` | Latest message in the partition |
| `LAG` | Messages not yet consumed = backlog |
| `CONSUMER-ID` | Which worker pod owns this partition |

### Interpreting lag

| LAG | Status |
|---|---|
| 0–5 | Healthy — consumer is keeping up |
| 5–50 | Light backlog — normal during burst |
| 50–500 | Growing — worker may be slow or restarting |
| > 500 | Problem — scale workers or investigate crash |

```bash
# Watch lag in real time
watch -n 5 "kubectl exec -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-consumer-groups.sh \
  --bootstrap-server localhost:9092 \
  --describe --group todo-worker 2>/dev/null"
```

## Consume messages live (tail the topic)

```bash
kubectl exec -it -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic todo-events \
  --from-beginning
```

Press `Ctrl+C` to stop. Each line is one message (JSON event from the outbox).

---

## Check total messages in topic

```bash
kubectl exec -n basesource kafka-0 -- \
  /opt/kafka/bin/kafka-run-class.sh kafka.tools.GetOffsetShell \
  --broker-list localhost:9092 \
  --topic todo-events \
  --time -1 2>/dev/null
```

Output:
```
todo-events:0:1252
todo-events:1:980
todo-events:2:762
```
Sum = 2994 total messages ever produced to this topic.

## Disk usage

```bash
kubectl exec -n basesource kafka-0 -- df -h /var/lib/kafka/data
```

Kafka retains messages by default for 7 days — disk grows over time.
**Alert when:** > 70% full.

## Kafka logs (broker health)

```bash
kubectl logs -n basesource kafka-0 -f | grep -E "ERROR|WARN|started|leader"
```

Healthy startup looks like:
```
[KafkaServer] started (kafka.server.KafkaServer)
[LeaderEpochFileCache] Completed load of leader epoch cache
```
